# 02. Object-Oriented Design (LLD) Masterclass

Object-Oriented Design (OOD), also known as Low-Level Design (LLD), is a critical part of the software engineering interview. It tests your ability to transform ambiguous requirements into a structured, maintainable, and scalable class-based architecture.

## 🎯 Learning Objectives
- Master the **SOLID** principles of software design.
- Apply common **Design Patterns** (Singleton, Strategy, Observer, Factory, etc.).
- Learn to create professional **UML Class Diagrams**.
- Practice designing complete systems like a Vending Machine or an ATM.

## 🛠️ The OOD Design Process
1. **Clarify Requirements**: Ask questions to define the scope.
2. **Identify Core Objects**: Extract nouns from the problem statement.
3. **Define Relationships**: Determine how objects interact (Is-a vs Has-a).
4. **Apply Patterns**: Use design patterns to handle complexity.
5. **Draw the Diagram**: Visualize the architecture using UML.
